# 견적서 웹 앱 MVP PRD 메타 프롬프트

## 🎯 메타 프롬프트 목적
Notion에서 관리하는 견적서 데이터를 웹 애플리케이션으로 제공하는 MVP의 PRD 문서를 자동 생성합니다.

## 📋 입력 정보 템플릿

아래 정보를 제공하면 PRD 문서가 자동으로 생성됩니다.

```markdown
## 프로젝트 기본 정보
- 프로젝트명: [e.g., 견적서 관리 웹 앱]
- 클라이언트명: [견적 대상 고객명]
- 견적 대상: [e.g., 웹 개발, 앱 개발, 디자인]

## 견적서 항목
### 항목 1
- 분류: [e.g., 기획/설계, 개발, QA, 배포]
- 설명: [상세 설명]
- 예상 시간/단가: [시간 or 금액]
- 수량: [개수]
- 소계: [금액]

### 항목 2
- [위와 동일 형식]

## 부가 정보
- 총 예상 기간: [e.g., 4주]
- 결제 조건: [e.g., 50% 선금, 50% 준공시]
- 유효 기간: [e.g., 2026-07-28]
- 특수 조건: [선택사항]
```

## 🔧 PRD 생성 프롬프트 (Claude에 전달)

```
당신은 경험 많은 IT 프로젝트 매니저이자 테크 라이터입니다.
아래 견적서 정보를 바탕으로 웹 애플리케이션 MVP의 PRD 문서를 작성하세요.

## 📌 작성 기준

### 1. 개요 (Overview)
- 프로젝트의 목표 (Why)
- 사용자 (클라이언트)가 얻을 수 있는 가치
- MVP 범위 (무엇을 포함하고 제외하는가)

### 2. 사용자 요구사항 (User Requirements)
#### 2.1 주요 사용자
- 클라이언트 (견적서 조회)
- 관리자 (견적서 작성/수정) - 2차 고려사항

#### 2.2 핵심 기능
| 기능 | 설명 | 우선순위 |
|-----|------|---------|
| 견적서 조회 | 클라이언트가 웹에서 견적서 내용 확인 | P0 |
| PDF 다운로드 | 견적서를 PDF 파일로 다운로드 | P0 |
| 반응형 디자인 | 모바일/태블릿/PC 모두 지원 | P1 |
| 인쇄 최적화 | 인쇄 레이아웃 최적화 | P1 |

### 3. 기술 스택 & 아키텍처
```
프론트엔드: 
- React + TypeScript (또는 Vanilla JS)
- Tailwind CSS / Bootstrap (스타일링)
- html2pdf / PDFKit (PDF 생성)

백엔드:
- Node.js + Express (또는 Fastify)
- Notion API (데이터 소스)
- 또는 JSON/CSV 데이터 소스

배포:
- Vercel / Netlify (프론트엔드)
- 또는 Heroku (풀스택)
```

### 4. 데이터 구조
```json
{
  "quotation": {
    "id": "string",
    "clientName": "string",
    "projectName": "string",
    "items": [
      {
        "category": "string",
        "description": "string",
        "quantity": "number",
        "unitPrice": "number",
        "subtotal": "number"
      }
    ],
    "totalAmount": "number",
    "currency": "KRW",
    "issueDate": "YYYY-MM-DD",
    "expiryDate": "YYYY-MM-DD",
    "paymentTerms": "string",
    "specialConditions": "string"
  }
}
```

### 5. 페이지 구성 (MVP)

#### 5.1 견적서 상세 페이지 (메인)
```
┌─────────────────────────────────────┐
│         [회사 로고/이름]             │
├─────────────────────────────────────┤
│  견적서                   2026-06-28 │
├─────────────────────────────────────┤
│  클라이언트: [고객명]                │
│  프로젝트: [프로젝트명]              │
├─────────────────────────────────────┤
│  항목  │ 수량 │ 단가  │ 소계        │
├─────────────────────────────────────┤
│ [항목1] │  1  │ 1,000,000 │ 1,000,000 │
│ [항목2] │  2  │  500,000  │ 1,000,000 │
├─────────────────────────────────────┤
│                    합계: 2,000,000   │
├─────────────────────────────────────┤
│  결제 조건: 50% 선금, 50% 준공시    │
│  유효 기간: 2026-07-28             │
├─────────────────────────────────────┤
│  [PDF 다운로드] [인쇄]               │
└─────────────────────────────────────┘
```

#### 5.2 기능 상세
**PDF 다운로드**
- html2pdf 라이브러리 사용
- A4 크기, 가로 방향
- 로고, 회사 정보, 견적 내용 포함
- 파일명: quotation_[프로젝트명]_[날짜].pdf

**인쇄 기능**
- 브라우저 기본 인쇄 (Ctrl+P)
- Print CSS 최적화

### 6. 성공 기준 (Success Metrics)
- [ ] 견적서가 웹에서 즉시 조회 가능
- [ ] PDF 다운로드 버튼 클릭 시 5초 내 생성
- [ ] 모바일에서 가독성 유지
- [ ] 로딩 시간 < 2초
- [ ] 클라이언트 만족도 >= 4/5

### 7. MVP 범위 (In-Scope)
✅ 견적서 데이터 조회 (Notion/JSON)
✅ PDF 다운로드
✅ 반응형 레이아웃
✅ 기본 인증 (토큰/링크 기반)

### 8. 제외 사항 (Out-of-Scope)
❌ 견적서 작성/편집 (관리자 대시보드)
❌ 복수 견적서 조회
❌ 결제 연동
❌ CRM 통합

### 9. 일정 & 우선순위
**Phase 1 (주 1-2): 프로토타입**
- 정적 견적서 페이지
- PDF 다운로드 구현

**Phase 2 (주 3): 완성 & 배포**
- Notion API 연동 또는 JSON 데이터 소스
- 반응형 디자인 완성
- 배포 & 테스트

### 10. 위험 요소 & 대응
| 위험 | 영향 | 대응 방안 |
|-----|------|---------|
| Notion API 응답 지연 | 페이지 로딩 느림 | 캐싱 또는 정적 데이터 사용 |
| PDF 생성 실패 | 사용자 불편 | 폴백: 인쇄 권장 |
| 모바일 레이아웃 깨짐 | 클라이언트 경험 저하 | 미디어 쿼리 및 테스트 |

```markdown
## 프로젝트 기본 정보

## 견적서 항목
### 항목 1 - 기획 및 설계

- 설명: React + TypeScript 기반 UI 개발
- 예상 시간: 120시간
- 단가: 60,000원/시간

### 항목 3 - 백엔드 개발

# 사내 예약 시스템 MVP PRD

ABC 의료원의 사내 예약 시스템은 직원들이 회의실, 장비, 시간을 효율적으로 예약할 수 있는 웹 애플리케이션입니다.

## 2. 사용자 요구사항
- 직원: 쉬운 예약 인터페이스
- 관리자: 예약 현황 모니터링

## 3. 기술 스택
- 프론트엔드: React + TypeScript + Tailwind CSS
- 백엔드: Node.js + Express + MongoDB
- 배포: Vercel + Railway
...
