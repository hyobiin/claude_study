# 노션을 활용한 맛집 블로그 MVP PRD

## 1. 개요

### 1.1 프로젝트 목표
Notion을 CMS(Content Management System)로 활용하여 비개발자도 쉽게 블로그 글을 작성하고 관리할 수 있는 맛집 블로그 웹 애플리케이션을 구축합니다.

### 1.2 사용자 가치
- **블로거**: Notion 인터페이스에서 친숙한 방식으로 글 작성 가능
- **방문자**: 반응형 웹에서 깔끔한 UI로 맛집 정보 확인 가능
- **관리**: 별도의 관리자 대시보드 없이 Notion에서 직접 관리

### 1.3 MVP 범위
- Notion API 연동 기반 블로그 시스템
- 글 목록 및 상세 페이지
- 카테고리별 필터링
- 기본 검색 기능
- 반응형 디자인 (모바일/태블릿/PC)
- SEO 최적화

## 2. 사용자 요구사항

### 2.1 주요 사용자

| 사용자 | 역할 | 필요 기능 |
|--------|------|---------|
| 블로거 | 콘텐츠 작성자 | Notion에서 글 작성/수정/삭제 |
| 방문자 | 글 열람자 | 글 목록 조회, 카테고리 필터링, 검색 |

### 2.2 핵심 기능 요구사항

| 기능 | 설명 | 우선순위 |
|-----|------|---------|
| Notion API 연동 | 블로그 데이터를 Notion에서 자동으로 가져오기 | P0 |
| 글 목록 페이지 | 최근 글 목록을 그리드/리스트 형태로 표시 | P0 |
| 글 상세 페이지 | 개별 글의 전체 내용 표시 | P0 |
| 카테고리 필터링 | 카테고리별로 글 필터링 | P0 |
| 검색 기능 | 제목/태그로 글 검색 | P1 |
| 반응형 디자인 | 모바일/태블릿/PC 모두 지원 | P0 |
| 글쓰기 기능 | 사이트에서 직접 글 작성 (웹 폼) | P2 |
| 댓글 기능 | 글에 댓글 추가/조회 | P2 |

### 2.3 사용자 플로우

**방문자 기본 플로우**
```
홈 → 글 목록 조회 → 글 상세 보기 → 다른 글 탐색
```

**필터링 플로우**
```
홈 → 카테고리 선택 → 카테고리별 글 목록 → 글 상세
```

## 4. 데이터 구조

### 4.1 Notion 데이터베이스 스키마

#### 테이블: Blog Posts

| 칼럼명 | 타입 | 설명 | 필수 |
|--------|------|------|-----|
| Title | Title | 글 제목 | ✓ |
| Category | Select | 카테고리 (한식, 중식, 양식 등) | ✓ |
| Tags | Multi-select | 태그 (분위기, 가격대 등) | ○ |
| Published | Date | 발행 날짜 | ✓ |
| Status | Select | 상태 (초안, 발행됨, 보관) | ✓ |
| Content | Page | 본문 내용 | ✓ |
| Featured Image | Files & Media | 대표 이미지 | ○ |
| Summary | Rich Text | 짧은 요약 (100자 이내) | ○ |

### 4.2 API 응답 데이터 구조

#### GET /api/posts (글 목록)

```json
{
  "posts": [
    {
      "id": "uuid",
      "title": "맛있는 한정식당",
      "category": "한식",
      "tags": ["분위기좋음", "가족식사"],
      "publishedDate": "2026-06-28",
      "summary": "강남 최고의 한정식당 리뷰",
      "featuredImage": "https://...",
      "slug": "delicious-korean-restaurant"
    }
  ],
  "total": 42
}
```

#### GET /api/posts/[slug] (글 상세)

```json
{
  "post": {
    "id": "uuid",
    "title": "맛있는 한정식당",
    "category": "한식",
    "tags": ["분위기좋음", "가족식사"],
    "publishedDate": "2026-06-28",
    "summary": "강남 최고의 한정식당 리뷰",
    "content": "<p>본문 내용...</p>",
    "featuredImage": "https://...",
    "slug": "delicious-korean-restaurant",
    "readTime": 5
  }
}
```

#### GET /api/search (검색)

```json
{
  "results": [
    {
      "id": "uuid",
      "title": "맛있는 한정식당",
      "snippet": "강남 최고의 한정식당..."
    }
  ]
}
```

## 5. 페이지 구성 (MVP)

### 5.1 홈 페이지 (/)

**목적**: 최근 발행 글 목록 표시

**구성 요소**:
```
┌─────────────────────────────────┐
│        [Logo] 맛집 블로그        │ Header
├─────────────────────────────────┤
│ 📍 위치 | 🏷️ 카테고리 필터      │ Navigation
├─────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐    │
│  │  이미지  │  │  제목     │    │ Post Card
│  │          │  │  요약     │    │ (Grid Layout)
│  │          │  │ 카테고리  │    │
│  └──────────┘  └──────────┘    │
│                                  │
│  ┌──────────┐  ┌──────────┐    │
│  │  이미지  │  │  제목     │    │
│  │          │  │  요약     │    │
│  │          │  │ 카테고리  │    │
│  └──────────┘  └──────────┘    │
├─────────────────────────────────┤
│  [← 이전]      [다음 →]         │ Pagination
└─────────────────────────────────┘
```

**기능**:
- 최근 글 12개 표시 (페이지네이션)
- 카테고리별 필터링
- 검색바 (선택사항: P1)

### 5.2 글 상세 페이지 (/posts/[slug])

**목적**: 개별 글의 전체 내용 표시

**구성 요소**:
```
┌─────────────────────────────────┐
│        [Logo] 맛집 블로그        │ Header
├─────────────────────────────────┤
│          [대표 이미지]            │
├─────────────────────────────────┤
│  제목: 맛있는 한정식당           │
│  카테고리: 한식                  │ Metadata
│  태그: #분위기좋음 #가족식사      │
│  발행: 2026-06-28              │
├─────────────────────────────────┤
│  본문 내용...                   │
│  (Notion에서 가져온 HTML)       │
├─────────────────────────────────┤
│  다음 글: [다음 글 제목]  →      │ Navigation
│  이전 글: ← [이전 글 제목]       │
└─────────────────────────────────┘
```

**기능**:
- 글 본문 표시 (Notion 콘텐츠 렌더링)
- 메타데이터 표시 (카테고리, 태그, 발행일)
- 이전/다음 글 네비게이션
- 목차 (선택사항: P1)

### 5.3 카테고리 페이지 (/category/[category])

**목적**: 특정 카테고리의 글 목록 표시

**구성 요소**:
- 홈 페이지와 동일하되, 선택된 카테고리로 필터링

## 7. MVP 범위 (In-Scope)

### ✅ 포함 사항

1. **Notion API 연동**
   - Notion 데이터베이스에서 자동으로 글 데이터 가져오기

   - 카테고리별 글 분류
   - 필터 UI

4. **기본 스타일링**
   - Tailwind CSS + shadcn/ui로 깔끔한 UI
   - 브랜드 색상 정의

5. **반응형 디자인**
   - Mobile First 접근
   - 모든 화면 크기에서 가독성 확보

6. **SEO 기본**
   - Next.js 메타 태그 (title, description)

## 10. 위험 요소 & 대응책

| 위험 | 영향 | 확률 | 대응 방안 |
|-----|------|------|---------|
| Notion API 응답 지연 | 페이지 로딩 느림 | 중 | ISR(Incremental Static Regeneration)로 캐싱, 타임아웃 설정 |

| 모바일 레이아웃 깨짐 | 사용자 경험 저하 | 낮음 | 충분한 모바일 테스트, 반응형 테스트 도구 활용 |
